# LLM-MM-Agent: HMML 数学建模知识库

> Path: `clean version/LLM-MM-Agent/MMAgent/HMML/`
> **重要程度**: ⭐⭐⭐ 核心知识源

HMML（Hierarchical Mathematical Modeling Library）是系统的核心知识库，提供数学建模领域的专业知识。采用三层层级结构组织 98+ 建模模式，是系统进行问题分析和方法选择的基础。

---

### `__init__.py` ⭐
包初始化文件，将 HMML 目录标记为 Python 包。

---

### `HMML.md` ⭐⭐⭐
**HMML 核心文档**，以 Markdown 格式呈现的完整知识库。采用 3 层层级结构组织建模方法：

1. **域（Domain）**：顶层分类，如优化、预测、分类、聚类、评价、仿真等 6 大领域
2. **子域（Subdomain）**：中层细分，如优化下的线性规划、整数规划、非线性规划、动态规划
3. **方法节点（Method Node）**：具体方法，包含方法名称、描述、适用场景、数学公式（LaTeX）、参数说明、实现要点、示例案例

每个方法节点提供详细的建模指导，包括问题特征识别、方法选择依据、参数设置建议、结果解释方法、常见陷阱提示。这是系统的核心知识源，支持问题感知检索（根据问题类型匹配方法）和解决方案感知检索（根据已有方案推荐相关方法）。是 LLM-MM-Agent 区别于其他系统的关键创新。

---

### `HMML.json` ⭐⭐⭐
**HMML 的 JSON 格式表示**，包含层级结构化的建模方法元数据。每个方法节点包含丰富的结构化信息：
- `name`：方法名称（中英文）
- `description`：方法描述（功能、原理、特点）
- `domain`：所属域（优化、预测、分类等）
- `subdomain`：所属子域（线性规划、回归分析等）
- `applicability`：适用场景（问题特征、数据要求）
- `formula`：数学公式（LaTeX 格式）
- `parameters`：参数说明（含义、取值范围、调参建议）
- `examples`：示例案例（应用场景、效果说明）
- `embedding`：文本嵌入向量（用于相似度匹配计算）

JSON 格式便于程序化访问、解析和嵌入相似度计算，是检索算法的直接数据源。与 HMML.md 内容一致但格式不同，分别服务于人类阅读和机器处理。

## HMML 核心特性

1. **3 层层级结构**：Domain → Subdomain → Method Node，从一般到特殊，便于导航和理解
2. **98+ 建模模式**：覆盖数学建模的主要类型（优化、预测、分类、聚类、评价、仿真）
3. **问题感知检索**：根据问题类型和特征自动检索和推荐合适的方法
4. **解决方案感知检索**：根据已有解决方案检索相关方法，支持方法组合
5. **嵌入相似度匹配**：使用文本嵌入计算方法与问题的语义相似度，超越关键词匹配
6. **Top-K 检索**：返回 top_method_num（默认 6）个最相关方法，提供多个选择

---

## HMML 应用场景

- **问题分析阶段**：根据问题描述自动检索候选方法，为后续建模提供选择
- **建模阶段**：提供详细的建模指导和公式，辅助构建数学模型
- **方法选择**：基于问题特征和数据特点推荐最合适的方法
- **知识扩展**：作为数学建模的教学参考，帮助理解各类方法
- **方法组合**：检索多个相关方法，支持组合建模策略
