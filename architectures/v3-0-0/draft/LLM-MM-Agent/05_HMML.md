### `HMML.md`
**HMML（Hierarchical Mathematical Modeling Library）核心文档**，以Markdown格式呈现98+数学建模模式的层级知识库。采用3层层级结构组织：

1. **域（Domain）**：如优化、预测、分类、聚类、评价等
2. **子域（Subdomain）**：如优化下的线性规划、整数规划、非线性规划
3. **方法节点（Method Node）**：具体的建模方法，包含方法描述、适用场景、数学公式、实现要点

每个方法节点提供详细的建模指导，包括问题特征、方法选择依据、参数设置建议、结果解释。这是系统的核心知识源，支持问题感知和解决方案感知的方法检索。

---

### `HMML.json`
HMML的JSON格式表示，包含层级结构化的建模方法元数据。每个方法节点包含：
- `name`: 方法名称
- `description`: 方法描述
- `domain`: 所属域
- `subdomain`: 所属子域
- `applicability`: 适用场景
- `formula`: 数学公式（LaTeX格式）
- `parameters`: 参数说明
- `examples`: 示例案例
- `embedding`: 文本嵌入向量（用于相似度匹配）

JSON格式便于程序化访问和嵌入相似度计算。

---

### `HMML` 核心特性

1. **3层层级结构**：Domain → Subdomain → Method Node
2. **98+建模模式**：覆盖优化、预测、分类、聚类、评价等主要数学建模类型
3. **问题感知检索**：根据问题类型和特征检索合适的方法
4. **解决方案感知检索**：根据已有解决方案检索相关方法
5. **嵌入相似度匹配**：使用文本嵌入计算方法与问题的相似度
6. **Top-K检索**：返回top_method_num（默认6）个最相关方法

---

### `HMML` 应用场景

- **问题分析阶段**：根据问题描述检索候选方法
- **建模阶段**：提供详细的建模指导和公式
- **方法选择**：基于问题特征推荐最合适的方法
- **知识扩展**：作为数学建模的教学参考
