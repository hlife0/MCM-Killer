# LLM-MM-Agent: 项目根目录

> Path: `clean version/LLM-MM-Agent/`
> **重要程度**: ⭐⭐⭐ 项目配置与文档中心

本目录包含 LLM-MM-Agent 项目的核心配置文件、依赖定义、文档说明和启动脚本。作为项目的顶层目录，它定义了系统的整体运行参数，管理项目依赖关系，提供用户文档和开发指南，并包含项目启动所需的各类初始化文件。开发者在此目录进行项目配置和环境搭建。

---

## 配置文件

### `config.yaml` ⭐⭐⭐
**主配置文件**，控制整个系统的运行参数和行为模式。定义了核心迭代轮次配置（problem_analysis_round: 3, problem_modeling_round: 3, task_formulas_round: 5）、任务数量限制（num_tasks: 5, num_charts: 5）、方法检索数量（top_method_num: 6）、路径配置（paths.root_data: "MMBench", paths.output_root: "MMAgent/output"）、调试模式开关（debug_mode）以及代码执行超时设置。这是系统运行的单一配置入口，任何对系统行为的调整都需要通过此文件进行。修改此文件可以显著影响系统的性能、输出质量和资源消耗。

### `requirements.txt` ⭐⭐⭐
**Python依赖包清单**，列出项目所需的所有第三方库及其版本要求。包含数据处理库（pandas, numpy, scipy）、可视化工具（matplotlib, seaborn, plotly）、LLM接口（openai, anthropic）、日志系统（loguru）、科学计算（sklearn, statsmodels）、文档处理（markdown, pdfkit）等核心依赖。通过此文件可以使用 `pip install -r requirements.txt` 一键安装所有依赖，确保环境一致性。对于开发和部署环境，这是必需的依赖管理文件。

## 启动脚本

### `run.py` ⭐⭐⭐
**启动引导脚本**，自动设置Python路径环境并启动主程序。处理常见的导入路径问题（确保能正确导入MMAgent模块），验证环境配置，提供友好的错误提示，确保从任何目录都能正确运行系统。对于不熟悉Python路径管理的用户，这是推荐的使用方式，可以避免因路径配置错误导致的导入失败问题。

### `__init__.py` ⭐
**包初始化文件**，将目录标记为Python包，允许使用import语句导入模块。虽然当前项目主要通过run.py或main.py启动，但此文件为将来的模块化扩展和包导入提供了基础支持。是Python包管理的标准文件。

## 其他文件

### `LICENSE` ⭐
**开源许可证文件**，规定项目的使用、修改、分发条款。定义了用户可以如何使用本代码、是否需要署名、是否可以用于商业用途、修改后的代码如何分发等法律条款。使用者应当阅读并遵守许可证要求，以确保合法使用项目代码。

### `.DS_Store` ⭐
**Mac系统自动生成的文件**，存储文件夹的显示属性（如图标大小、列表视图位置等）。此文件对项目功能无影响，通常应加入.gitignore避免提交到版本控制系统。在不同操作系统间传输项目时可能产生冲突，建议删除或忽略。

### `.gitattributes` ⭐
**Git属性配置文件**，定义文件的换行符处理（LF vs CRLF）、差异比较工具、合并策略等。对于跨平台开发团队协作非常重要，确保在不同操作系统（Windows/Linux/macOS）间代码格式一致性，避免因换行符差异导致的版本控制问题。

### `logo.png` ⭐
**项目Logo图片**，用于文档、演示文稿、网站展示等场合。作为项目的视觉标识，增强项目识别度和专业性。在README、论文、PPT等场景中使用，提升项目形象。

### `demo.mp4` ⭐⭐
**演示视频**，展示系统运行的完整过程和用户交互流程。包括如何启动系统、输入问题、观察中间结果、查看最终报告等关键步骤。对于新用户来说是快速了解系统能力的最佳方式，也可以用于学术演示、项目汇报等场景。视频质量直接影响用户的第一印象和学习效率。
