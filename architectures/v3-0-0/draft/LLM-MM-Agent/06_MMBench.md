## Dataset 目录 (`dataset/`)

存储历年 MCM/ICM 竞赛题目的原始数据文件，按年份和题目编号组织。从 2000 年到 2025 年，涵盖各类题型（连续型、数据洞察型、跨学科型）。

### 关键年份数据 ⭐⭐⭐

#### `2006_C/` ⭐⭐⭐
2006 年 C 题数据，关于 WHO 成员国的人口、健康、经济数据。包含 9 个 XLS 文件（人口、出生率、生育率、预期寿命、HIV/AIDS、收入、疫苗接种率等）。是典型的多元数据分析问题，适合测试系统处理多源数据的能力。

#### `2019_C/` ⭐⭐⭐
2019 年 C 题数据，关于毒品管制和人口统计数据。包含大容量毒品报告数据（MCM_NFLIS_Data.xlsx）和美国社区调查数据（7 年数据，每个目录包含数据文件、元数据、注释）。是大数据分析和时序建模的典型问题，测试系统的数据处理和时序分析能力。

#### `2020_C/` ⭐⭐
2020 年 C 题数据，关于亚马逊产品评论的文本分析。包含三个 TSV 文件（吹风机、微波炉、安抚奶嘴的评论数据）。是文本挖掘和情感分析的典型问题，测试系统处理非结构化文本数据的能力。

#### `2021_C/` ⭐⭐
2021 年 C 题数据，关于胡蜂检测和图像识别。包含数据集 Excel 文件、图像清单 Excel 文件、200+ 张胡蜂图片（JPG/PNG 格式）。是计算机视觉和图像识别问题，测试系统的图像处理能力。

#### `2024_C/`, `2025_C/` ⭐⭐⭐
最新年份的数据文件，代表最新的题目趋势和难度水平。是系统性能评估的主要测试集。

## Problem 目录 (`problem/`)

存储历年题目的 JSON 格式陈述，是系统的输入数据源。

### 文件命名规则 ⭐⭐
`{YEAR}_{LETTER}.json`，如 `2000_A.json`、`2024_C.json`。标准化的命名便于批量处理和自动加载。

### 题目类型
- **A/B**：MCM 连续型题目（数学建模）
- **C**：MCM 数据洞察题目（数据分析）
- **D/E/F**：ICM 跨学科题目（ interdisciplinary）
- **Y/Z**：其他类型题目

### JSON 结构 ⭐⭐⭐
每个题目文件包含结构化的题目信息：
- `year`：年份
- `type`：题目类型（MCM/ICM）
- `letter`：题目编号（A/B/C/D/E/F）
- `title`：题目标题
- `description`：问题描述（完整文本）
- `requirements`：具体要求（列表形式）
- `data_files`：相关数据文件（路径列表）

这种结构化格式便于程序化解析和处理，是系统自动化运行的基础。

## Evaluation 目录 (`evaluation/`)

### `__init__.py` ⭐
包初始化文件。

### `model.py` ⭐⭐⭐
**评估模型实现**，定义评估指标和评分方法。包括相关性评估（是否回答问题）、准确性评估（计算正确性）、完整性评估（是否覆盖要求）、创新性评估（方法新颖性）。提供量化的评分标准。

### `prompts.py` ⭐⭐
**评估提示词**，用于生成评估报告和质量评分。引导 LLM 分析解决方案的优缺点、识别不足之处、提供改进建议。

### `run_evaluation.py` ⭐⭐⭐
**单次评估运行器**，执行单个任务或题目的评估。加载题目和解决方案、调用评估模型、生成评估报告、保存评分结果。是评估系统的主入口。

### `run_evaluation_batch.py` ⭐⭐
**批量评估运行器**，批量执行多个任务或题目的评估。支持批量模式（一次评估多个题目）、并行执行（提升效率）、汇总统计（生成总体报告）。适合大规模系统测试。

### `utils.py` ⭐⭐
**评估工具函数**，提供评估相关的辅助功能。包括数据加载、结果解析、报告生成、统计分析等。

## Example Solution 目录 (`example_solution/`)

### `example1.json`, `example2.json` ⭐⭐
**示例解决方案文件**，展示标准解决方案的格式和结构。包括问题分析、方法选择、模型构建、结果展示、结论总结等完整流程。是系统学习和模仿的参考样本。

## Processed Data 目录 (`processed_data/`)

### `processed_data/` ⭐⭐
**存储原始数据集的处理版本**，包括清洗后的数据、特征工程后的数据、预处理的缓存。避免重复处理原始数据，提升系统运行效率。是数据处理的中间结果仓库。

## MMBench 核心价值

1. **历史数据覆盖**：25 年（2000-2025）的竞赛题目和数据
2. **题型多样性**：连续型、数据型、跨学科型、图像型、文本型
3. **标准化格式**：统一的 JSON 题目格式和 CSV/XLS 数据格式
4. **评估框架**：完整的评估指标和评分系统
5. **基准测试**：为系统性能评估提供标准测试集
